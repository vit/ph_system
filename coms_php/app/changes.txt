0.1.0:
15.03.2005

Добавлена новая почтовая подсистема (несколько таблиц
в структуре данных). 

В интерфейсе менеджера конференции - функции работы с почтой.
Можно создавать и редактировать письма, содержащие 
подстановочные символы ({$PIN}, {$PAPTITLE} и т.п.).
Список получателей писем составляется функцией-продюсером
(из предопределенного набора функций).

Также в структуру заложена возможность определения
администратором системы шаблонов писем, которые
могут использоваться менеджерами конференций
(пока не задействована).

Кроме того, сделаны таблицы для хранения "синхронных"
писем (отправляемых при определенных событиях).
Эта функция пока также не реализована.
Синхронные письма (автописьма) посылаются прежним
методом (их тексты хранятся в файлах).

В разделе "Papers" менеджера добавлена кнопка
"Copy all final decisions and sections from editors'".

Добавлены отчеты "Papers by final decision".

0.0.4:
18.01.2005

В автописьмах кроме подстановочных переменных
с информацией о пользователе могут использоваться
переменные, специфические для конкретного письма.
Имена и значения переменных передаются массивом
в функцию mail_send_now.

Обратный адрес для автописем определяется глобальной
переменной $FROMEMAIL, заданной в файле конфигурации.

Добавлено новое автописьмо:
при загрузке статьи регистратор может запросить
подтверждение успешной загрузки по e-mail,
установив соответствующий флажок в форме загрузки.
Будет сообщен номер статьи и название конференции.

На странице "Papers" менеджера конференции список
статей отображается постранично - 20 статей на странице.

Добавлен отчет менеджеру конференции:
"Papers by registrator's countries" -
"Список статей, сгруппированный по странам регистраторов".


0.0.3:
14.01.2005

Допустимый размер файла увеличен:
File size not greater than 2Mb
В форме загрузки файла атрибут MAX_FILE_SIZE содержит
значение 2096000 байт (что чуть меньше, чем 2Mb).
Возможность загрузки файлов такого объема не гарантирована,
поскольку допустимый размер запроса может ограничиваться
на промежуточных прокси-серверах.

Размер шрифтов "шапки" еще уменьшен.

В пользовательском интерфейсе теперь вместо слова
"Subject" используется "Section".
В программах и структуре базы данных остался прежний термин.

Раздел "Manager tool" переименован в "Manager office".

В "Editor's office" и "Reviewer's office" перечень статей
и секций теперь отображается без помощи таблиц.
